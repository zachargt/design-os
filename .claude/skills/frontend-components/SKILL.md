# Frontend Components

## When to use this skill

- When creating new UI components or refactoring existing ones
- When defining component props, interfaces, or APIs
- When designing reusable components for multiple contexts
- When implementing component composition patterns
- When managing component state and deciding where state should live
- When splitting large components into smaller, focused pieces
- When building component libraries or design systems
- When documenting component usage, props, and examples
- When working with component files (.jsx, .tsx, .vue, .svelte)
- When encapsulating component implementation details
- When establishing naming conventions for components
- When creating configurable components with sensible defaults

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle frontend components.
