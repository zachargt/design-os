# Global Coding Style

## When to use this skill

- When writing any new code in the project
- When refactoring existing code for better readability
- When naming variables, functions, classes, or files
- When creating small, focused functions with single responsibilities
- When removing dead code, unused imports, or commented-out blocks
- When extracting common logic into reusable functions
- When ensuring consistent indentation and formatting
- When applying the DRY (Don't Repeat Yourself) principle
- When choosing descriptive names that reveal intent
- When working with any code file regardless of language (.js, .py, .rb, .java, .ts, etc.)
- When configuring code formatters or linters
- When avoiding backward compatibility logic unless required

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle global coding style.
