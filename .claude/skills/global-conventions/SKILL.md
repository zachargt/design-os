# Global Conventions

## When to use this skill

- When organizing project files and directory structure
- When writing or updating README files and documentation
- When creating commit messages for version control
- When managing project dependencies (package.json, requirements.txt, Gemfile)
- When configuring environment variables and settings
- When setting up feature flags for incomplete features
- When maintaining changelogs or release notes
- When establishing code review processes
- When defining testing requirements before merging
- When ensuring secrets and API keys are not committed
- When documenting architecture overview and contribution guidelines
- When working with configuration files (.env.example, config files)

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle global conventions.
