# Frontend Accessibility

## When to use this skill

- When creating new UI components or modifying existing ones
- When building forms with input fields, labels, and validation
- When implementing interactive elements (buttons, links, dropdowns)
- When adding keyboard navigation or focus management
- When creating navigation menus or site headers
- When implementing modals, dialogs, or overlays
- When choosing color combinations and ensuring contrast ratios
- When adding alternative text to images or icons
- When using ARIA attributes for complex components
- When structuring document headings (h1-h6) hierarchically
- When working with component files (.jsx, .tsx, .vue, .svelte, .html)
- When testing components with screen readers
- When implementing single-page application focus management

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle frontend accessibility.
