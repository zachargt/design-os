# Backend Migrations

## When to use this skill

- When creating new database migration files
- When altering existing database schemas (adding/removing tables or columns)
- When modifying table structures or constraints
- When creating or dropping database indexes
- When writing rollback/down methods for migrations
- When separating schema changes from data migrations
- When implementing zero-downtime database deployments
- When working with migration files (e.g., *.migration.ts, *_migration.rb, migrations/*.py)
- When managing database version control and migration history
- When creating indexes on large tables with concurrent options

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle backend migrations.
