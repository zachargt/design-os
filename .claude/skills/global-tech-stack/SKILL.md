# Global Tech Stack

## When to use this skill

- When choosing technologies or libraries for new features
- When adding new project dependencies
- When configuring build tools, bundlers, or compilers
- When setting up testing frameworks or CI/CD pipelines
- When writing framework-specific code (React, Vue, Rails, Django, etc.)
- When working with the project's database or ORM
- When integrating third-party services (auth, email, monitoring)
- When ensuring consistency with existing technology choices
- When documenting the tech stack in README or configuration files
- When working with package managers (npm, pip, bundler, etc.)
- When making decisions about CSS frameworks or UI libraries
- When setting up development, staging, or production environments

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle global tech stack.
